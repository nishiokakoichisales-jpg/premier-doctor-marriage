# LP制作プロジェクト — Claude Code 運用ルール

## このファイルについて
Claude Codeが自動的に読み込む指示書です。LP制作時のルール・制約・進め方を定義しています。

---

## 制作者プロフィール
- 非エンジニア（HTML/CSSはなんとなく分かるレベル）
- 自分がデザインの最終判断者（クライアント承認なし）
- 対象業種: 富裕層男性×女性のマッチング系サービス（結婚相談所・交際クラブ・パパ活アプリ等）
- 1サービスにつき男性向けLP・女性向けLPの両方を作ることがある
- 画像素材はMiriCanvas AIで毎回生成する
- 制作目標時間: 4時間以内

## 制作環境
- エディタ: VSCode + Claude Code
- 確認: GitHub Pages（制作段階）
- 本番: 独自ドメイン・サーバー
- フォーム: Zoho Forms / Zoho Bookings / LINE連携システム（案件による）

---

## 絶対に守るルール

### 1. 実装順序（これを破らない）
```
Phase 1: 仕様確認（コードを書く前に必ず行う）
Phase 2: 素材の確認（全素材が揃っていることを確認）
Phase 3: スマホ版を完成させる（FV → 残りセクション）
Phase 4: スマホ版のOKが出てからPC版に着手
Phase 5: PC版はメディアクエリで追加（スマホ版を壊さない）
Phase 6: 最終確認 → コミット → デプロイ
```

### 2. 仕様確認フェーズ（Phase 1で必ずやること）
コードを書く前に以下を確認し、不足があれば質問する：
- [ ] 参考サイトURLを確認したか
- [ ] FVの手書きラフまたは配置イメージを受け取ったか
- [ ] テキスト原稿の大枠を受け取ったか
- [ ] 使用する画像素材が全てassets/に揃っているか
- [ ] ロゴ画像があるか（既存 or 生成が必要か）
- [ ] フォームの種類（Zoho Forms / Bookings / LINE）を確認したか
- [ ] 追従CTAバーが必要かどうか確認したか
- [ ] 男性向けか女性向けか確認したか
- [ ] セクション構成を提案し、承認を得たか

### 3. コード規約
- CSSは1ファイル（styles.css）にまとめる
- **使わなくなったCSSは即座に削除する**（コメントアウトで残さない）
- CSS変数（:root）を定義したら実際に使う。使わない変数は定義しない
- **同じセレクタを2箇所で定義しない**
- コメントに書いた計算値（例: 「80px × 1.5 = 120px」）は変更時に必ず更新する
- inline style属性は使わない（CSSファイルに書く）

### 4. ファイル管理
- ファイル名は英数字ハイフン区切り（例: couple-main.jpg）
- 透過版は `-t` サフィックス統一（例: logo-emblem-t.png）
- **使わないファイルは即座に削除する**
- 重複ファイル（-2サフィックスなど）は作らない
- 二重拡張子（.png.png）は絶対に作らない

### 5. 指示への対応ルール
- **指示された箇所だけを修正する**。他を勝手に変えない
- **推測で機能や装飾を追加しない**
- 不明点はコードを書く前に質問する
- 「いい感じに」「適切に」と言われたら、具体案を2〜3個提示して選ばせる
- サイズ変更指示は「現在値 → 新値」を明示してから実行する
- スマホ版完成後にPC版を触るとき、**スマホ版に影響がないことを必ず確認する**

### 6. Git運用
- 以下のタイミングで必ずコミットする：
  - FV完成時
  - スマホ版全体完成時
  - PC版完成時
  - 大きな修正の後
- コミットメッセージは日本語でOK
- A/Bテストの2案はブランチで管理する（pattern-a, pattern-b）

### 7. 画像
- 全画像は納品前にTinyPNG等で圧縮する（1枚500KB以下目安）
- OGP画像（ogp.jpg）とfavicon（favicon.ico）を忘れない
- assets/内は必要なファイルのみ。不要ファイルは都度削除

## セクション構成の提案ルール
セクション構成は案件ごとに異なるため、毎回以下の手順で決める：
1. 参考サイトを分析してセクション構成を抽出
2. サービス内容に合わせた構成案を2パターン提示
3. ユーザーが選択・修正してから実装に入る

## テキスト原稿の作成ルール
ユーザーから大枠のみ受け取り、Claudeが仕上げる：
1. ユーザーからサービス概要・訴求ポイント・ターゲットを聞く
2. キャッチコピー案を3パターン提示
3. 各セクションの文言を下書きして確認を取る
4. 承認後に実装する（実装しながらテキストを変えない）

## ディレクトリ構成（標準）
```
プロジェクト名/
├── CLAUDE.md          ← このファイル
├── index.html         ← LP本体
├── styles.css         ← スタイル
├── script.js          ← JS
├── assets/            ← 使用中の素材のみ
│   ├── （画像ファイル群）
│   └── （ロゴ・バッジ等）
├── docs/              ← 制作ドキュメント
│   ├── 素材プロンプト.md
│   └── テキスト原稿.md
└── 納品物/            ← 完成版コピー（最終のみ）
```

## 制作開始時の標準フロー

### Step 1: 準備（制作者がやること）
```
□ 参考サイトURLを用意
□ サービスの概要をメモ（何のサービスか・ターゲット・訴求ポイント）
□ FVの手書きラフを描いてスマホで撮影
□ ロゴ画像があれば assets/ に配置
□ 男性向けか女性向けか決める
```

### Step 2: Claude Codeに初回指示
```
以下の初回プロンプトを貼る（別ファイル参照）
```

### Step 3: 素材生成
```
□ Claudeが出した素材プロンプトでMiriCanvasで画像生成
□ 全画像をassets/に正しい名前で保存
□ TinyPNGで圧縮
□ 「素材全部揃いました」とClaudeに伝える
```

### Step 4: 実装→確認ループ
```
□ スマホ版FV → スクショ確認 → OKならコミット
□ スマホ版残りセクション → 確認 → OKならコミット
□ PC版調整 → 確認 → OKならコミット
□ 修正は3つ以上まとめてから指示する
```

### Step 5: 納品
```
□ 未使用ファイル削除
□ 画像圧縮確認
□ GitHub Pagesデプロイ
□ URL共有
```
